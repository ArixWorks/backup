# راهنمای راه‌اندازی فروشگاه (Go-Live)

این سند مرحله‌به‌مرحله توضیح می‌دهد چطور پروژه را روی یک سرور واقعی (VPS یا هر میزبان دیگر) با **دیتابیس تازه و بدون دیتای دمو** راه‌اندازی کنید.

---

## ۱. خلاصه‌ی استراتژی

دیتای دمو (کاربران Sara/Reza/Niloofar، موجودی‌های الکی، محصولات و مزایده‌های نمونه) **هرگز نباید به production منتقل شود**. به‌جای کپی‌کردن دیتابیس دمو:

1. یک دیتابیس کاملاً تازه می‌سازید.
2. فقط **داده‌ی ساختاری** (ارزها، نشان‌ها، ماموریت‌ها) را seed می‌کنید.
3. اولین ورود شما با تلگرام، حساب شما را **به‌صورت خودکار ادمین** می‌کند.
4. محصولات و قیمت‌های واقعی را از داشبورد ادمین وارد می‌کنید.

> دستور `pnpm db:seed` (سیدِ دمو) را **روی production اجرا نکنید** — آن فقط برای توسعه است. برای production فقط از `pnpm db:seed:prod` استفاده کنید.

---

## ۲. چه چیزی می‌ماند و چه چیزی پاک می‌شود

| ساختاری (می‌ماند) | تراکنشی/دمو (پاک می‌شود) |
|---|---|
| `Setting` / `BotSetting` | کاربران، کیف‌پول‌ها، موجودی‌ها |
| `Currency` / `ExchangeRate` | محصولات، فروش ثابت، انبار |
| `Badge` / `Mission` | مزایده‌ها، پیشنهادها، سفارش‌ها |
| `AlertRule` | واریز/برداشت/استرداد، دفترکل |
|  | امتیازها، نشان‌های کاربر، نوتیفیکیشن‌ها |

> تنظیمات اپ (درصد کش‌بک، سطوح VIP، تم و ...) در کد مقدار پیش‌فرض امن دارند، پس نیازی به seed ندارند و از داشبورد قابل تنظیم‌اند.

---

## ۳. متغیرهای محیطی لازم (`.env`)

روی سرور این‌ها را تنظیم کنید:

| متغیر | الزامی | توضیح |
|---|---|---|
| `DATABASE_URL` | ✅ | رشته‌ی اتصال PostgreSQL (مثلاً Neon یا Postgres روی VPS) |
| `TELEGRAM_BOT_TOKEN` | ✅ | توکن بات از BotFather |
| `ADMIN_TELEGRAM_IDS` | ✅ | آی‌دی عددی تلگرام شما (چند تا با کاما). اولین ورود = ادمین خودکار |
| `AUTH_SECRET` | ✅ | کلید امضای نشست. با `openssl rand -base64 32` بسازید |
| `CRON_SECRET` | ✅ | کلید محافظت endpointهای cron (مزایده/مانیتورینگ) |
| `NEXT_PUBLIC_APP_URL` / `APP_URL` | ✅ | آدرس عمومی اپ (مثلاً `https://shop.example.com`) |
| `REDIS_URL` | توصیه‌شده | برای کش/قفل/ریت‌لیمیت (Upstash یا Redis روی VPS) |
| `RESEND_API_KEY` | اختیاری | ارسال ایمیل |
| `SENTRY_DSN` | اختیاری | گزارش خطا |

> آی‌دی عددی تلگرام خود را می‌توانید با بات `@userinfobot` بگیرید.

## ۴. مراحل استقرار روی سرور

```bash
# ۱) نصب وابستگی‌ها
pnpm install

# ۲) مهاجرت اسکیما روی دیتابیس تازه + seed ساختاری (یک دستور)
pnpm go-live
#  معادل:  prisma migrate deploy && tsx scripts/production-seed.ts

# ۳) ساخت نسخه‌ی production
pnpm build

# ۴) اجرای اپ (پشت یک reverse proxy مثل Nginx + SSL)
pnpm start

# ۵) اجرای سرویس WebSocket عملیات (در صورت استفاده از داشبورد Ops زنده)
pnpm ops:ws
```

برای پایداری، اپ و `ops:ws` را با یک process manager مثل **pm2** یا **systemd** اجرا کنید تا بعد از ری‌استارت بالا بمانند.

---

## ۵. زمان‌بندی Cron (مزایده/مانیتورینگ)

در پلن Hobby ورسل cronهای زیردقیقه‌ای رد می‌شوند، برای همین از یک زمان‌بند بیرونی استفاده می‌کنیم (مثل **cron-job.org**). هر دقیقه این دو آدرس را با هدر `Authorization: Bearer <CRON_SECRET>` صدا بزنید:

```
POST/GET  https://<APP_URL>/api/v1/cron/tick
POST/GET  https://<APP_URL>/api/v1/cron/monitor
```

> اگر روی VPS هستید، می‌توانید به‌جای آن از crontab سیستم استفاده کنید (هر دقیقه `curl` با همان هدر).

---

## ۶. اولین ورود به‌عنوان ادمین

1. مطمئن شوید `ADMIN_TELEGRAM_IDS` آی‌دی شما را دارد.
2. بات را در تلگرام `/start` بزنید و Mini App را باز کنید.
3. حساب شما به‌صورت خودکار **ADMIN** می‌شود (ارتقا فقط؛ هیچ‌وقت downgrade نمی‌کند).
4. وارد داشبورد ادمین شوید و موارد زیر را انجام دهید:
   - نرخ ارز واقعی را در بخش مالی به‌روز کنید (نرخ‌های پیش‌فرض صرفاً اولیه‌اند).
   - محصولات و قیمت‌های واقعی را اضافه کنید.
   - کانال‌های عضویت اجباری، کش‌بک، سطوح VIP و تم را تنظیم کنید.

---

## ۷. اگر دیتابیس از قبل دیتای دمو دارد (Factory Reset)

اگر به‌جای دیتابیس تازه، همان دیتابیس دمو را دارید و می‌خواهید پاکش کنید:

```bash
CONFIRM_RESET=ERASE pnpm db:reset   # همه‌ی دیتای کاربر/تراکنش را پاک می‌کند، ساختار را نگه می‌دارد
pnpm db:seed:prod                   # تضمین وجود داده‌ی ساختاری (idempotent)
```

> این دستور بدون `CONFIRM_RESET=ERASE` اجرا نمی‌شود تا اشتباهی پاک نشود. **این عمل برگشت‌ناپذیر است** — قبل از اجرا حتماً بکاپ بگیرید.

## ۸. چک‌لیست نهایی قبل از باز کردن فروشگاه

- [ ] `DATABASE_URL` به دیتابیس production اشاره می‌کند (نه دمو)
- [ ] `pnpm go-live` بدون خطا اجرا شد
- [ ] `ADMIN_TELEGRAM_IDS` ست شده و ورود ادمین تست شد
- [ ] `AUTH_SECRET` و `CRON_SECRET` مقادیر تصادفی و امن دارند
- [ ] دامنه + SSL فعال است و `NEXT_PUBLIC_APP_URL` درست است
- [ ] Mini App و دامنه در BotFather ثبت شده‌اند
- [ ] زمان‌بند cron هر دقیقه `tick` و `monitor` را می‌زند
- [ ] نرخ ارز واقعی در داشبورد به‌روز شد
- [ ] حداقل یک محصول واقعی اضافه و یک خرید آزمایشی انجام شد
- [ ] بکاپ‌گیری منظم دیتابیس فعال است
